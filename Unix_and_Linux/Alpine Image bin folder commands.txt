nice is a command and a system call in Unix-like operating systems that allows users to adjust the priority of a process. The priority level determines how much CPU time a process gets in relation to other processes. Lower priority values mean higher priority for the process, and higher priority values mean lower priority.

The nice command is used to launch a command with a specified priority, or it can be used to change the priority of an already running process.

Here's a basic overview of how the nice command works:


nice [options] [command [arguments...]]

nice -n 10 ./my_program
This example launches the my_program executable with a nice value of 10. A higher nice value indicates a lower priority.

Options:
-n or --adjustment: Specifies the nice value. Values range from -20 (highest priority) to 19 (lowest priority). The default nice value is 10.
-20 to 19: Specifies the nice value directly, without using the -n option.

Changing Priority of Running Processes:
If you want to change the priority of a process that is already running, you can use the renice command. This command requires the process ID (PID) of the target process.

renice -n 10 -p 1234
This example changes the priority of the process with PID 1234 to a nice value of 10.

Viewing Nice Values:
The ps command can be used to view the nice values of running processes. The nice value is typically displayed in the NI (nice) column.

ps aux
The nice command is useful when you want to control the priority of processes, especially in a multi-user environment or when you want to ensure that certain tasks do not monopolize the CPU. Keep in mind that adjusting process priorities might require superuser (root) privileges in some cases. Additionally, some systems use a different mechanism called the Completely Fair Scheduler (CFS) or other schedulers, which may affect how priorities are managed.
